✅️ functional_requirements:
	➡️ R1: Keyword Input
		The application shall accept a keyword from the user as input.
		The keyword shall be limited to alphanumeric characters and have a maximum length of 100 characters.
	➡️ R2: Video URL Input
		The application shall accept a YouTube video URL from the user as input.
	➡️ R3: Video Metrics Collection
		The application shall collect video metrics from YouTube based on the provided keyword or video URL.
		The metrics collected shall include video title, channel name, number of views, number of comments, etc.
	➡️ R4: Comment Collection
		The application shall collect all comments associated with the specified video.
	➡️ R5: CSV Output
		The application shall generate a CSV file containing the collected video metrics and comments data.
		The user shall be able to download the CSV file via a button on the web page.
	➡️ R6: User-Friendly Interface
		The application shall have a user-friendly interface that allows users to easily input their keyword or video URL and view the collected data.
	➡️ R7: Error Handling
		If the user input is invalid, the application shall display an error message and ask the user to enter a valid keyword or video URL.
		If there is an error collecting the video metrics or comments data, the application shall display an error message and terminate the process.
		If there is an error generating the CSV file or providing it for download, the application shall display an error message and terminate the process.


✅️ nonfunctional_requirements:
